# Sosaia Web Client

Next.js + Three.js voxel city viewer. Renders precomputed mesh data from the [sosaia-preprocessing-rust](https://github.com/your-org/sosaia-preprocessing-rust) pipeline.

## Getting Started

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Architecture

### Rendering Pipeline

The web client loads **precomputed mesh data** generated by the Rust preprocessing pipeline. No runtime meshing occurs in the browser — vertex buffers are uploaded directly to the GPU.

```
Rust pipeline (.mesh.bin) → fetch → parseMeshBin() → THREE.BufferGeometry → GPU
```

### LOD (Level of Detail)

Three LOD levels per chunk, auto-switched by Three.js based on camera distance:

| Level | File suffix    | Distance   | Detail        |
|-------|---------------|------------|---------------|
| LOD0  | `.mesh.bin`   | 0-400m     | Full detail   |
| LOD1  | `.lod1.mesh.bin` | 400-1000m | ~7x fewer verts |
| LOD2  | `.lod2.mesh.bin` | 1000m+    | ~40x fewer verts |

### Physics Collision

Raw block data (`.bin` files from `chunks/`) is loaded into `VoxelGrid` for collision detection. Meshes handle rendering, blocks handle physics.

## Data Layout

Place preprocessed city data in `public/`:

```
public/
  chunks/<city-name>/
    manifest.json
    0_0.bin, ...
  meshes/<city-name>/
    manifest.json
    0_0.mesh.bin
    0_0.lod1.mesh.bin
    0_0.lod2.mesh.bin
    ...
```

## Key Files

- `src/components/World.tsx` — Main world loader (mesh + physics loading, game loop)
- `src/game/meshLoader.ts` — Parses `.mesh.bin` binary format into Three.js geometry with LOD
- `src/game/chunkLoader.ts` — Loads raw block `.bin` files for physics collision
- `src/game/voxelGrid.ts` — 3D voxel grid for physics/collision queries
- `src/game/playerPhysics.ts` — Player movement + collision detection
- `src/game/materials.ts` — Material ID to Three.js material mapping
- `src/game/vehicles.ts` — Vehicle system (airplane, magic carpet, hot air balloon)
- `src/game/network.ts` — WebSocket multiplayer client
- `src/game/config.ts` — Runtime config (render settings, debug flags)

## Mesh Binary Format (.mesh.bin)

```
Header (10 bytes):
  magic:       u32 LE = 0x534F534D ("SOSM")
  version:     u16 LE = 1
  chunk_x:     i16 LE
  chunk_z:     i16 LE
Material count: u16 LE
Per material:
  material_id:    u16 LE
  vertex_count:   u32 LE
  index_count:    u32 LE
  positions:      [f32 LE; vertex_count * 3]
  normals:        [f32 LE; vertex_count * 3]
  colors:         [f32 LE; vertex_count * 3]  (AO baked)
  indices:        [u32 LE; index_count]
```

## Controls

- **WASD / Arrow keys** — Move
- **Space** — Jump
- **Shift** — Sprint
- **V** — Toggle vehicle mount/dismount
- **1/2/3** — Switch vehicle type (airplane, magic carpet, hot air balloon)
- **E** (in vehicle) — Boost
- **Mouse** — Look around (pointer lock)
